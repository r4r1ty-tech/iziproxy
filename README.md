# IziProxy 🚀

**IziProxy** — это автоматизированный инструмент на C# (.NET 10) для быстрого и безопасного развертывания прокси-сервера Xray (Sing-box/V2ray) на вашем личном VPS. Утилита берет чистый сервер на Debian/Ubuntu и за минуту превращает его в пуленепробиваемый прокси-сервер с настроенным клиентом.

## ✨ Ключевые возможности

- **Современный протокол**: Использует связку **VLESS + XHTTP + REALITY**. Это один из самых скрытных протоколов на сегодняшний день, который маскирует ваш трафик под обычный HTTPS-запрос к популярному сайту (например, Microsoft или Apple).
- **Умный автоподбор SNI (домена маскировки)**: Скрипт не просто ставит заглушку. Он автономно пингует список трастовых доменов с самого VPS, проверяет поддержку HTTP/2 и TLS 1.3, измеряет задержку (`time_connect`, `time_appconnect`) и выбирает самый быстрый домен специально под ваш сервер.
- **Обход блокировок по RU-сегменту**: В `config.json` встроены правила маршрутизации (`geosite:ru`, `geoip:ru`), которые отправляют трафик к отечественным сайтам (Госуслуги, банки, Кинопоиск) **напрямую**, мимо прокси. Это спасает от блокировок по безопасности и сохраняет высокую скорость.
- **Ускорение сети (TCP BBR)**: Автоматически включает алгоритм контроля перегрузок BBR от Google в ядре Linux, что радикально уменьшает пинг и повышает пропускную способность.
- **Интеллектуальная работа с портами**: Ищет свободный порт (сначала проверяет 443 и 8443, если заняты — берет рандомный из диапазона 10000-50000) и самостоятельно настраивает брандмауэр UFW (закрывая старые дыры и открывая порт для прокси + 22 для SSH).
- **Динамическая криптография**: Все ключи безопасности (x25519 Private/Public keys, UUID, ShortID) генерируются уникально на самом сервере в момент установки.
- **Поддержка sudo / root**: Программа умеет исполнять команды как от прямого `root`, так и от обычного пользователя (используя `sudo` с передачей пароля).

## 📂 Структура проекта и как это работает

Вся логика разделена на две среды: управление осуществляется с локальной машины (C#), а тяжелая работа и конфигурации — на самом VPS (Bash).

1. **`Program.cs` / `SSH.cs`**: Локальный оркестратор. Подключается к VPS по логину и паролю, загружает bash-скрипты по SFTP.
2. **`VDS_setup/MainInstall.sh`**: Базовый бутстрап. Проверяет ОС (строго Debian/Ubuntu-like), ставит пакеты (curl, jq, ufw, openssl), активирует BBR и устанавливает ядро Xray.
3. **`XrayConfigParams.cs`**: Запрашивает у установленного Xray генерацию ключей (x25519) и случайных идентификаторов. Fetch-ит гео-локацию сервера через `ipinfo.io`.
4. **`VDS_setup/Deploy.sh` (Самый умный скрипт)**:
   - Поднимает порт.
   - Сканирует домены для REALITY, учитывая доступность TLS 1.3, отсеивает локальные IP и дает скоринг.
   - Записывает выбранный SNI и порт в конфигурацию с помощью `jq`.
5. **`VlessLinkGenerator.cs`**: Берет все сгенерированные данные, гео-метку, IP и формирует готовую ссылку (`vless://...`), которую можно сразу скопировать в клиент (Nekobox, V2rayNG, Hiddify).

## 🚀 Как начать

### Требования
- Установленный **.NET 10.0 SDK**.
- Виртуальный сервер (VDS/VPS) под управлением **Ubuntu** или **Debian**.
- Включенная авторизация по паролю для SSH (если у вас только ключи, сервер может отбросить подключение с ошибкой `No suitable authentication method (publickey)`).

### Запуск

Откройте терминал, перейдите в папку проекта и введите:

```bash
dotnet build
dotnet run
```

Утилита попросит вас ввести:
1. **IP-адрес сервера** (или домен).
2. **Имя пользователя** (например, `root` или `ubuntu`).
3. **Пароль**.

После этого скрипт всё сделает сам. На выходе в консоли (через 1-2 минуты) появится строка вида:
`vless://<uuid>@<ip>:<port>?type=xhttp&security=reality&pbk=<key>&fp=chrome&sni=<best-domain>&sid=<short-id>&path=/xh-query#IziProxy_VDS`

Просто скопируйте эту ссылку в свой клиент.

## 🛠 Устранение неполадок (Troubleshooting)

- **`No suitable authentication method found to complete authentication (publickey).`**
  **Решение:** На сервере отключен вход по паролю (стоит `PasswordAuthentication no` в `/etc/ssh/sshd_config`). Временно включите его или настройте, либо дождитесь реализации входа по SSH-ключам (.pem / .ppk).
- **`jq not found, skip config read/update`**
  **Решение:** По какой-то причине `MainInstall.sh` не смог установить пакет `jq`. Проверьте, работает ли на сервере `apt-get update`.
- **`SNI selection failed`**
  **Решение:** VPS находится в очень плохой сети или забанен IP, утилита `curl` не смогла достучаться ни до одного домена из встроенного списка. Вы можете передать свой список доменов в текстовом файле.
- **Всё работает, но медленно:**
  Убедитесь, что провайдер не режет UDP-трафик. Xray поверх XHTTP маскируется очень хорошо, но качество связи зависит от маршрутов между вашим провайдером и дата-центром VDS.
